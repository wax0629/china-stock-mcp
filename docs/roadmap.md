# China Stock MCP - 未来发展建议文档

> 版本：v1.0  
> 状态：路线图（非承诺）  
> 用途：在 v1.0 落地后，指导项目长期演进与生态扩展

---

## 1. 三阶段总览

```
┌─────────────┐   ┌──────────────┐   ┌───────────────┐
│  Phase A    │   │   Phase B    │   │   Phase C     │
│  立足 (0-3月)│ → │ 扩张 (3-9月) │ → │ 平台化 (9-18月)│
```

每个阶段都对应"做什么 / 为什么 / 衡量指标"，不要把后期目标提前。

## 2. Phase A：立足期（0-3 个月）

### 2.1 核心目标
**让"懂 A 股的开发者 / 投资者"都知道有这个项目，并愿意装上试用。**

### 2.2 必做事项

#### 工程层面
- 完成设计文档中 v1.0 的 10 个核心工具
- 端到端 demo 视频 / GIF（3 个典型场景）
- 单元测试覆盖率 ≥ 70%，CI 自动跑
- PyPI 发布、`uvx china-stock-mcp` 一键运行
- 文档站：使用 Mintlify / Docusaurus / mdBook

#### 推广层面
- 提交至以下列表/平台：
  - `punkpeye/awesome-mcp-servers`
  - `appcypher/awesome-mcp-servers`
  - `mcpservers.org`
  - Smithery
  - Hugging Face Spaces（做一个 Web demo）
- 内容产出（这一步比代码更重要）：
  - 知乎/掘金技术长文：`如何用 30 行代码给 Claude 接上 A 股能力`
  - 小红书/即刻短帖：3 个 GIF 演示
  - Bilibili 5-10 分钟演示视频
  - Twitter/X 英文版（吸引海外开发者）

#### 社区层面
- 开 GitHub Discussions，鼓励用户提"想要的工具"
- 建一个微信群 / Discord，承载早期用户
- 每周一次"用户问答日"，亲自回答前 5 个 issue

### 2.3 衡量指标（Phase A 出口标准）
- GitHub stars ≥ 200
- PyPI 月下载 ≥ 500
- 至少 3 篇外部博客提到该项目
- 至少 5 个真实用户的使用反馈
- 没有 P0 级 bug 超过 7 天未修

## 3. Phase B：扩张期（3-9 个月）

### 3.1 核心目标
**从"能用"走到"好用"，建立差异化护城河，让别人难以复制。**

### 3.2 工具与数据扩展

#### B-1：港股 / 美股扩展
- 港股：完整支持（用 akshare 港股接口）
- 美股：通过 Yahoo Finance / Alpha Vantage 接入，但定位是"辅助"
- 加 `currency` 参数，自动换算

#### B-2：可转债 / 期货 / 期权
- 可转债：到期收益率、转股价值、溢价率
- 期货主连：螺纹钢、原油、贵金属
- 期权：希腊字母、隐含波动率
- 这是个人投资者用得到、但市面工具又散的领域

#### B-3：另类数据
- 公告 NLP 摘要：调用 LLM 把长公告变摘要
- 研报库：聚合公开卖方研报标题与评级
- 投资者关系问答（互动易/上证 e 互动）：搜索关键词
- 高管增减持、股东户数变动
- ESG 数据

#### B-4：量化能力
- `backtest_strategy(symbol, strategy_dsl, start, end)`：内置常见策略 DSL（双均线、RSI 反转）
- `calc_correlation(symbols, period)`：相关性矩阵
- `calc_max_drawdown(symbol, period)`
- `simulate_portfolio(holdings)`：组合回测

### 3.3 协议与生态

#### B-5：MCP Resources 与 Subscriptions
- 收盘后市场总览 → resource（每日推送给订阅的 Agent）
- 自定义自选股监控 → resource，AI 可"挂单"提醒条件

#### B-6：A2A 协议接入
跟进 Google A2A 协议，让该 MCP 能作为"金融 Agent"被其他 Agent 调用，而不只是工具。

#### B-7：MCP Skills（FastMCP 3.x 特性）
打包"投研工作流"为 Skill：用户一键下载 `china-stock-research-skill`，Claude 自动激活全套相关工具与 prompt。

### 3.4 体验提升

#### B-8：可视化输出
- 在返回 Markdown 中嵌入 SVG/ASCII 图表（K 线、柱状图）
- 提供可选的 HTML 报告导出（一份 PDF/HTML 投研报告）

#### B-9：个性化记忆
- 引入 user profile：用户的风险偏好、自选股、研究兴趣
- AI 跨会话引用，比如"按你之前关注的低估值蓝筹风格筛选"

#### B-10：多语言
- 工具描述、错误信息、Markdown 输出支持中英双语
- 海外用户也能用，扩大受众

### 3.5 商业化探索（轻量）

> 原则：开源核心永远免费。商业化只动"高成本/高价值"边缘。

可选路径：
- **Pro 版数据**：接入付费数据源（Wind 接口、Choice），按月订阅
- **托管服务**：streamable-http 模式的多用户托管，免去本地配置
- **企业咨询**：帮券商/资管接入 MCP，定制工具
- **Sponsorship**：GitHub Sponsors / 爱发电

### 3.6 衡量指标
- GitHub stars ≥ 2000
- PyPI 月下载 ≥ 5000
- 至少 1 个第三方项目把它列为依赖
- 工具数量 ≥ 25
- 形成稳定的核心贡献者（≥ 2 人）

## 4. Phase C：平台化期（9-18 个月）

### 4.1 核心目标
**从一个 MCP Server，演进为以它为核心的金融 AI 产品矩阵。**

到这个阶段，MCP 本身已经成熟稳定，要思考"基于这个底座能做出什么用户产品"。

### 4.2 上层产品方向

#### C-1：每日盯盘 Agent（推荐先做）
- 调度器：每天 9:30 / 11:30 / 14:00 / 15:00 自动跑
- Agent：自动调用 MCP 工具，生成市场快讯/自选股动态/异动提醒
- 推送：微信（企业微信群机器人）/ 邮件 / Telegram
- 形态：开源 Python 项目，配置 yaml 即可

#### C-2：自动化投研工作流
- 多 Agent 协作：研究员 → 分析师 → 风控
- 输入一个公司名，输出一份完整的研报 PDF
- 用 LangGraph / CrewAI 编排
- 可独立成产品（Web App + 订阅）

#### C-3：财报解读 Agent
- 财报季自动抓取 → 摘要 → 异常检测 → 同业对比 → 结论
- 给非专业投资者用"人话"讲财报

#### C-4：基金经理对比工作台
- 输入两个基金经理，自动对比持仓变化、风格漂移、超额收益
- 个人投资者最缺这种工具

#### C-5：投资者情绪 Dashboard
- 整合：北向资金、龙虎榜、社交情绪（微博/雪球）、新闻情绪
- 一张大屏，每分钟更新
- 跟 v1.0 的"市场声化"思路结合：可以做成可挂在工位的环境监测

### 4.3 横向生态

#### C-6：MCP 中文金融生态联盟
联合相关方向的开发者，做一个"中文金融 MCP 标准集合"：
- china-stock-mcp（你的，行情/基本面）
- china-news-mcp（新闻/研报）
- china-economy-mcp（宏观经济）
- china-policy-mcp（政策法规）
- 大家的 MCP 互相调用，AI 拼装出复合能力

#### C-7：标准与认证
- 制定"中文金融 MCP 工具命名规范"（参考 OpenAI plugin 当年规范）
- 推动各家数据源统一字段命名
- 你的项目作为参考实现，影响力随之放大

#### C-8：教学内容
- B 站系列课：`如何用 MCP 构建你自己的 AI 投资助手`
- 出书 / 出电子书（GitBook 免费版）
- 在线讲座 / 沙龙

### 4.4 商业化加深（可选）

> 重要：是否商业化由你的精力和兴趣决定，不是必须。

如果走商业化：
- **SaaS 形态**：托管 MCP + 上层 Agent，按席位/调用量收费
- **数据增值**：买正版 Wind/Choice 数据，分发给个人订阅者（注意合规）
- **B 端**：给券商财富管理部门做"AI 投顾助手"

如果不走商业化：
- 维持开源公益项目地位，争取 Linux Foundation / Apache 孵化
- 个人通过演讲、咨询、写作变现

### 4.5 衡量指标
- GitHub stars ≥ 10k
- 月活 MCP 用户 ≥ 1 万
- 上层 Agent 产品中至少 1 个进入 ProductHunt 周榜
- 在 awesome-mcp-servers 头部 10 名内
- 形成清晰的"中文金融 AI 工具链"叙事

## 5. 长期定位思考

### 5.1 三种可能的终局

| 路径 | 描述 | 适合谁 |
|------|------|--------|
| **A：标准制定者** | 永远开源、追求广泛采用、商业化轻 | 你想专注技术影响力 |
| **B：商业化产品** | MCP 是基石，上层做付费 SaaS | 你愿意全职做创业 |
| **C：个人作品集** | 维持高质量但精力可控，作为简历亮点 | 你只想做副业、转行加分 |

**建议**：先按 A 走，过程中根据状态切换到 B 或 C，不要一开始定死。

### 5.2 持续监控的外部信号

- **MCP 协议演进**：Anthropic 路线图、FastMCP 主版本
- **A2A / A2UI**：Google / Anthropic 的 Agent 互操作协议
- **国内监管**：金融数据合规要求（这块要警惕）
- **数据源变化**：akshare 是否被官方限制、tushare 政策
- **竞品出现**：每月看一下有没有新的中文金融 MCP

### 5.3 长期风险与对冲

| 风险 | 对冲 |
|------|------|
| akshare 关停或被限 | 早期就做好适配层，多源融合 |
| 监管要求金融数据 API 资质 | 始终强调"个人研究学习"定位，不做交易撮合 |
| 大厂下场（同花顺/雪球）出官方 MCP | 用社区与中立性建立护城河，开源永远比闭源开放 |
| 你个人精力下降 | Phase A 末期就培养核心贡献者 |
| MCP 协议被新协议替代 | 适配层与协议层解耦，业务逻辑可移植 |

## 7. 反向清单：刻意不做的事

任何长期项目的关键不是"能做什么"，而是"不做什么"。

- ❌ **不做下单交易**。法律风险、责任风险都太高。
- ❌ **不做"荐股/推荐买卖"输出**。即使用户问，也只给数据不给结论。
- ❌ **不做高频实时数据**。订阅成本和合规成本高，不适合开源。
- ❌ **不做前端 UI**。MCP 客户端已经有 UI，自己做就是重复造轮子。
- ❌ **不做"All-in-one"金融平台**。坚守 MCP 工具层定位，谁都可以在上面盖楼。
- ❌ **不为短期 star 灌水**。慢慢做扎实，比刷一波更长远。

## 9. 给"未来某天动力下降的你"

如果某天你失去做下去的动力，问自己三个问题：

1. **是否还在解决真实问题？**  
   是 → 找一个用户聊半小时，听他怎么用，重燃方向感。  
   否 → 停下来，转向新方向，不丢人。

2. **是否还能学到新东西？**  
   是 → 加新功能，用新协议，做新尝试。  
   否 → 把维护交给社区，自己 step back。

3. **是否还在为你创造价值？**  
   是 → 哪怕慢，也值得继续。  
   否 → 优雅归档（把项目设为 archived，写一篇收尾文章），转身做下一个。

**项目可以结束，能力会留下；社区会变化，作品永远在。**
